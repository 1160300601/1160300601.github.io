  1. 模块化开发：
    浏览器只支持ES6的模块化，其他的需要使用webpack处理后才能在浏览器上使用
    模块化是将每个js文件作为一个模块导入或者导出，解决同时引用多个js文件时，变量重名的问题，到后来扩展为非js文件也能作为模块进行导入导出。

    1. export(导出)/import(导入)：
    2. export default/import：导入者可以对导出的内容重新命名,只能有一个default
    3. 统一全部导入：import * as name from './aaa.js',通过name.变量，name.函数，name.类取对应数据
    4. import后不写路径的话是从mode_modules中直接引用模块

  2. webpack: 开发代码 –> webpack处理下 –> 可部署。不然有些文件浏览器不支持开发代码的语法

    1. 核心：让我们能进行模块化开发，并帮助我们处理模块间的依赖关系。不仅打包JavaScript文件。css、image、json等都能当作模块来使用
    2. 用法：

  3. vue-cli
    需要注意的是，vue-cli是基于webpack的，与webpack不同，它尽量减少用户配置文件，将配置文件隐藏。尤其package.json中的"@vue/cli-service": "^4.4.0"管理了很多包

  4. ES6中函数的写法

  5. 网页开发的发展路程(很重要)

    1. JSP阶段：后端渲染：请求页面时，后端返回整个被渲染好的页面(html+css+js)，然后整体返回。后端路由：在后端将一个url和一个页面映射
    2. 前后端分离：
      • 后端：不进行渲染,只负责提供数据，后端分为静态资源服务器和提供API接口的服务器
      • 后端路由：仍然是后端路由阶段
      • 过程：输入url –> 从静态服务器请求(html+css+js) –> js由浏览器执行 –> ajax从API服务器请求数据，局部更新
    3. SPA页面：单页富应用，整个页面只有一个html页面
      • 后端：静态资源服务器只有一个 html+css+js，API服务器继续提供接口
      • 前端：前端路由，前端根据不同的url从得到的html+css+js中抽离出对应的页面
      • 过程：输入主页 –> 从静态服务器请求一个(html) –> 再请求其他url –> 从前面得到的资源抽离出来

  6. vue-router: vue的前端路由

    1. 实现的底层：url的hash更改，html5的history设置
    2. vue-router可把一个组件映射成一个url页面，每个 .vue 文件就是一个组件(包括html+css+js)
    3. router-link补充：
    4. 使用原生标签,不用router-link做路由跳转:
    5. vue-router为每个vue组件都绑定了注册的 router ,可以通过$router引用。动态路由中的$route和$router不同，$route表示现在活跃的路由。这还是因为所有组件都继承自vue的原型，VUE原型中的属性和方法，VUE组件都有。
    6. 路由的懒加载：路由中通常定义不同的页面，这页面最后会被打包到一个js文件中，如果一次请求，需要太长时间，可以使用懒加载方式解决
    7. 路由嵌套:比如在/home中分为/home/news 和 /home/message
    8. 动态路由：对于用户,/user/zhangsan 和 /user/lisi 组件一样，数据不一样，处理方式
    9. url 参数传递：除了上面的动态路由，也可以使用url中的query参数，query就是url中?之后的部分
    10. 路由守卫: 由一个路由跳转到另一个路由的过程，我们可以在里面实现一些动作
    11. keep-alive：组件创建后不被销毁。使用keep-alive的组件有两个方法 activated(激活)和deactivated(非激活)

  7. Promise 的使用

  8. vuex：对所有组件进行统一管理，使管理的组件之间能够获得其他组件的状态和属性。统一管理，便于维护

  9. axios

    • alias配置别名：默认 ‘@’:’src’，这样引用的时候，直接使用别名就行了

  11. .editorconfig: 配置一些代码风格等,使用的时候前面加个~

  12. Vue组件

    1. 如果请求数据时，可用外层的父组件请求数据，然后父组件将数据传递个子组件

    2. 组件之间的传递：父 –> 子，子–> 父

    3. 父–>子 使用 props进行传递：子组件定义props属性用于接收传来的数据，父组件使用 v-bind 将父组件的数据传给子组件

    4. 子传父：子组件使用this.$emit(“事件名称”,”事件参数”)向父组件发射事件，父组件使用 v-on 接收
